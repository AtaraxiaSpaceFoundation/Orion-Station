# Gemini repository guidance

Use `AGENTS.md` as the canonical entry point. Scoped `AGENTS.md` files refine rules by directory. Task instructions and references live in `.agents/skills/`; always-on constraints live in `.agents/rules/`.

Inspect current code and module manifests before proposing changes. Preserve server authority, prediction, localization, resource ownership, and test reporting. Do not treat paths from the old Orion repository as destinations.
