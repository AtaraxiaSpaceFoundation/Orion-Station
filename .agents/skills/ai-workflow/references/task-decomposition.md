# Task decomposition

Decompose around repository boundaries rather than arbitrary file counts.

Useful workstreams:

- ownership and dependency mapping;
- server authority and networking;
- client UI and presentation;
- prototypes, localization, and assets;
- persistence and migrations;
- tests and CI;
- port provenance and licensing;
- final integration review.

Give each subagent a concrete question, allowed paths, forbidden actions, expected evidence, and result format. Do not ask every agent to solve the entire task independently.
