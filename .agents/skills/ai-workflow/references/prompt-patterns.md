# Prompt patterns

A useful implementation prompt specifies:

- repository and branch;
- current architecture and relevant owners;
- required behavior;
- explicit exclusions;
- source material allowed for comparison;
- validation commands;
- Git and delivery constraints;
- expected final report.

A useful review prompt asks for concrete findings ordered by severity, with paths, reproduction or failure mode, and a minimal remediation. Avoid prompts that ask only whether the code is "good".
